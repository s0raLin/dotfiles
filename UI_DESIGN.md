# UI 设计说明

## 🎯 按钮功能重新设计

### 问题分析
原来的设计中存在功能重叠：
- TopNavbar: "保存更改" 
- Editor: "保存文件" + "应用配置"

这会让用户困惑，不知道应该点击哪个按钮。

### 新的交互逻辑

#### 📍 TopNavbar（全局操作）
- **导入配置**: 批量导入配置文件
- **全部应用**: 将所有修改过的配置文件一次性应用到系统

#### 📍 Editor（单文件操作）
- **撤销更改**: 撤销当前文件的修改
- **复制内容**: 复制当前文件内容到剪贴板
- **导出此文件**: 下载当前文件
- **保存到系统**: 将当前文件的修改应用到系统（只有修改时才可用）

### 🔄 状态管理

#### 文件修改状态
- **未修改**: 按钮显示"已保存"，灰色不可点击
- **已修改**: 按钮显示"保存到系统"，橙色可点击
- **文件名旁显示**: 修改状态用 `●` 标识

#### 全局状态
- TopNavbar 的"全部应用"按钮会检测是否有任何文件被修改
- 如果有修改，按钮变为绿色可点击状态

### 🎨 视觉设计原则

#### 颜色语义
- **灰色**: 不可用/已完成状态
- **橙色**: 警告/需要保存
- **绿色**: 成功/应用操作
- **蓝色**: 主要操作
- **红色**: 危险操作

#### 按钮层级
1. **主要操作**: 大按钮，明显颜色
2. **次要操作**: 中等按钮，中性颜色
3. **辅助操作**: 小按钮或图标按钮

### 📱 响应式考虑
- 小屏幕上隐藏按钮文字，只显示图标
- 保持重要操作的可访问性
- 使用 tooltip 提供操作说明
